# راهنمای راه‌اندازی Backend (نسخه اول)

سیستم مدیریت و ارزیابی متقاضیان استخدام — با Node.js + Express + Prisma + **SQLite**

> این نسخه با SQLite کار می‌کند: دیتابیس فقط **یک فایل** است و نیازی به نصب هیچ سرور دیتابیسی ندارید. بعداً برای تبدیل به PostgreSQL فقط دو خط در فایل `schema.prisma` تغییر می‌کند.

---

## ۱) پیش‌نیاز: فقط Node.js را نصب کنید

اگر Node.js را ندارید، از این آدرس نسخه **LTS** را دانلود و نصب کنید:

https://nodejs.org

برای مطمئن شدن از نصب، ترمینال (در ویندوز: **PowerShell** یا **Command Prompt**) را باز کنید و بنویسید:

```bash
node -v
```

اگر یک عدد مثل `v20.x.x` دیدید، یعنی نصب درست است.

## ۳) اجرای پروژه — فقط ۴ دستور

ترمینال را باز کنید و وارد پوشه `backend` شوید. سپس این دستورها را **یکی‌یکی** بزنید:

**قدم ۱ — نصب کتابخانه‌ها** (فقط یک‌بار لازم است):

```bash
npm install
```

**قدم ۲ — ساخت فایل دیتابیس و جدول‌ها:**

```bash
npx prisma migrate dev --name init
```

بعد از این دستور، فایل دیتابیس `dev.db` به‌صورت خودکار داخل پوشه `prisma` ساخته می‌شود.

**قدم ۳ — اجرای سرور:**

```bash
npm run dev
```

اگر پیام `🚀 سرور روی پورت 4000 اجرا شد` را دیدید، همه‌چیز درست است.
حالا در مرورگر آدرس زیر را باز کنید:

```
http://localhost:4000
```

باید پیام `Backend در حال اجراست ✅` را ببینید.

**قدم ۴ (اختیاری) — دیدن دیتابیس به‌صورت گرافیکی:**

```bash
npx prisma studio
```

یک صفحه در مرورگر باز می‌شود که می‌توانید همه جدول‌ها و داده‌ها را ببینید.

## ۴) بعداً چطور به PostgreSQL تبدیل کنم؟

در فایل `prisma/schema.prisma` فقط این قسمت را عوض می‌کنید:

```prisma
datasource db {
  provider = "postgresql"   // به‌جای "sqlite"
  url      = env("DATABASE_URL")
}
```

و در فایل `.env` آدرس دیتابیس PostgreSQL را می‌گذارید. کدهای Prisma تقریباً یکسان می‌مانند.

> نکته: در SQLite از `enum` و آرایه پشتیبانی نمی‌شود، برای همین در طراحی، این مقادیر به‌صورت `String`/`JSON` ذخیره شده‌اند. هنگام مهاجرت به PostgreSQL می‌توانید در صورت تمایل آن‌ها را به `enum` واقعی تبدیل کنید.

## جدول‌های دیتابیس (خلاصه)

| جدول | کاربرد |
|------|--------|
| `Employer` | کارفرما و احراز هویت (ورود، بازیابی رمز) |
| `JobPosition` | موقعیت شغلی + لینک اختصاصی (slug) |
| `Form` | فرم استخدام هر شغل |
| `Question` | سؤالات فرم (۸ نوع، با ترتیب قابل تغییر) |
| `ScoringRule` | قوانین امتیازدهی (هیچ امتیازی در کد ثابت نیست) |
| `RejectionRule` | قوانین رد خودکار |
| `Applicant` | متقاضی (بدون نیاز به ثبت‌نام) |
| `Application` | درخواست + امتیاز نهایی + دسته‌بندی + فیلدهای آماده AI |
| `Answer` | پاسخ‌های متقاضی به سؤالات |
